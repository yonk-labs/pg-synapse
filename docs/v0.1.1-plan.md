# pg_synapse v0.1.1 Implementation Plan

Closes the v0.1.0 known gaps. Priority order is binding: N1 (security and
integrity defects) ships before any new features.

## N1 - pgrx security and integrity hardening (P0)

These are defects in shipped code, not missing features.

### N1.1 SECURITY DEFINER + role grants
- All `synapse.*` functions marked `SECURITY DEFINER` with explicit
  `GRANT EXECUTE`: write/admin functions to `synapse_admin`, read/run
  functions (`execute`, `embed`, `*_list`, `execution_status`) to
  `synapse_user`.
- `REVOKE ALL ... FROM PUBLIC` on the schema; callers never touch
  `synapse.secrets` directly.
- Test: as `synapse_user`, `SELECT * FROM synapse.secrets` fails, but
  `synapse.execute(...)` succeeds; as a role with neither grant, admin
  functions fail.

### N1.2 SAVEPOINT-per-tool-call
- Wrap every tool dispatch inside `synapse.execute` in a SPI SAVEPOINT.
- On tool failure, roll back to the savepoint so a failing later tool call
  does not discard an earlier tool's committed writes.
- Test: agent with two tool calls where the second fails; first call's row
  survives, second's is rolled back, the execution still records the error.

### N1.3 Positional bind params for sql_query / sql_exec
- `SpiSqlExecutor` binds the `params` array via typed SPI parameters instead
  of rejecting them.
- Remove the "inline literals, do not use params" instruction from the
  example agent prompts; switch them to parameterized form.
- Test: `sql_exec('INSERT ... VALUES ($1,$2)', ['a','b'])` works and is
  injection-safe (a value containing `'); DROP TABLE` is treated as data).

## N2 - pgrx functional completeness (P1)

### N2.1 NUMERIC cost roundtrip + cost cap enforcement
- `cost_usd` and `cost_cap_usd` roundtrip as NUMERIC (not lossy f64/None).
- The pgrx host enforces the agent cost cap: `RAISE EXCEPTION` when the
  accumulated cost exceeds the cap; the execution row records
  `cost_cap_exceeded`.

### N2.2 Remaining SQL functions
Add: `execute_async` + `execution_status` (async path backed by a job row
and a background worker), `tool_call` (direct tool invocation for testing),
`agent_list`, `llm_profile_drop`, `embedding_profile_drop`, `secret_drop`,
`tool_register`, `tool_list`. Each with `SECURITY DEFINER` + the right grant.

### N2.3 Remaining GUCs + fallback resolution
- Implement the full GUC set from the design spec (10 GUCs), including
  `default_cost_cap_usd`, `default_embedding_profile`, `sidecar_url`, etc.
- `Runtime` / host resolves agent fields from the GUC default when the
  agent row leaves them NULL (llm_profile_main, timeout, max_iterations,
  cost_cap).

## N3 - llama-cpp provider + local granite demo (P1, explicit user ask)

### N3.1 pg-synapse-provider-llama-cpp
- Client speaking llama.cpp server (OpenAI-compatible mode primary).
- `download` feature: auto-fetch a GGUF (granite 3.x / granite-4 class,
  under 2B) from Hugging Face into `~/.cache/pg-synapse/models/`.
- Bonus: implement `EmbeddingProviderFactory` over llama.cpp `/v1/embeddings`.
- wiremock unit tests + `--features live-tests` gated live test.

### N3.2 examples/local-llama-agent
- Repeatable `run.sh`: downloads the GGUF, starts llama-cpp-server on a free
  port, registers the profile, runs an agent that reads/writes a table,
  asserts, tears down. Captured `EXAMPLE_OUTPUT.md`.

## N4 - Anthropic provider plugin (P2)

`pg-synapse-provider-anthropic`: messages API, `tool_use` / `tool_result`
content blocks, `cache_control` passthrough via `params`. wiremock + gated
live tests. Registered via `Runtime::builder().with_plugin(...)`.

## N5 - Sidecar binary host (P2)

### N5.1 pg-synapse-sidecar
axum server, `clap` CLI (`--port`, `--database-url`, `--admin-token`), the
12 v1 endpoints from decisions D7, `sqlx`-backed `ProfileSource` and
`SqlExecutor`, `X-PG-Synapse-Admin-Token` on `/v1/admin/*`.

### N5.2 sql/sidecar-install.sql
Template creating the same `synapse` schema, tables, roles, grants, and
SECURITY DEFINER functions that forward to the sidecar via
`pg_net.http_post`. `{{SIDECAR_URL}}` placeholder.

### N5.3 docker e2e
Dockerized Postgres on a free port (per decision D12) + sidecar; run the
sql-agent-readwrite workflow through the sidecar path; assert identical
behavior to the extension.

## N6 - Release v0.1.1 (gate)

Re-verify all example workflows live, CHANGELOG entry, workspace version
bump to 0.1.1, `cargo package` recheck, local `v0.1.1` tag.

## Discipline

Same rules as v0.1.0: clean-room (no private codebase refs), no em-dashes,
typed errors only, `fmt` + `clippy -D warnings` + `test` green before any
commit, subagent two-stage review per task, docker Postgres on a free port,
no external git push.
